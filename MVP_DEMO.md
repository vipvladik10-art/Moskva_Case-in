# MVP Demo — Метео-планировщик асфальтоукладки

Документ для жюри и команды: как запустить, что показать, что моковое, что реальное.

---

## 1. Быстрый запуск (без Docker)

Нужно: Python 3.11+ (на машине разработки используется 3.14) и Node.js 20+.

### Backend (терминал 1)

```powershell
cd backend
py -m venv .venv
.venv\Scripts\python.exe -m pip install fastapi "uvicorn[standard]" pydantic pydantic-settings httpx structlog PyYAML pytest pytest-asyncio
.venv\Scripts\python.exe -m uvicorn app.main:app --port 8000
```

API будет на <http://localhost:8000/docs> (Swagger).

### OpenWeatherMap (опционально, но желательно)

Реальная погода включается через локальный `.env` в корне проекта:

```powershell
OPENWEATHER_API_KEY=<ваш_ключ>
```

Ключ не коммитится: `.env` находится в `.gitignore`. Если ключ пустой, API недоступен
или лимит исчерпан, backend автоматически откатывается на mock-прогноз.

### Frontend (терминал 2)

```powershell
cd frontend
npm install
npm run dev
```

Откройте <http://localhost:5173>. Vite проксирует `/api` и `/ws` на backend.

> Для MVP БД и Docker не нужны: backend работает на in-memory fixtures из
> [data/fixtures](data/fixtures). Расчёты используют реальные формулы из техкарты.

## 2. Сценарий показа (3–5 минут)

### Шаг 1. Карта операций (15 сек)
- Открываем главную страницу `/`.
- Видим 3 АБЗ (жёлтые маркеры), 5 реальных участков М-11 в диапазоне
  `км 184+807 - 209+311` (зелёные маркеры), радар осадков поверх карты.
- В левом верхнем углу карты показаны статус радара RainViewer и источник погоды:
  `OpenWeatherMap` или `mock fallback`.
- В правой панели: «Демо-сценарий», «Открытые наряды ТО», «Журнал решений».

### Шаг 2. Карточка участка (40 сек)
- Кликаем по участку №2 «М-11, км 189+445 - 194+292».
- Показываем:
  - **тип смеси**, ширину, толщину, плотность;
  - **24-часовую таймлайн-полосу прогноза**: зелёные часы — ясно, оранжевые —
    холодно, красные — дождь. Контур обведён вокруг «зелёного окна»;
  - **виджет «Успеть до дождя»**: рекомендованный тоннаж с запасом 5 %, плюс
    объяснение, что ограничивает — АБЗ или укладчик;
  - **альтернативные АБЗ** с временем доставки.

### Шаг 3. Внезапный дождь (1 мин)
- Возвращаемся на карту.
- В правой панели выбираем «М-11, км 189+445 - 194+292» и нажимаем
  **«Запустить дождь»**.
- За 2–3 секунды:
  - маркер участка №2 становится красным и пульсирует;
  - в шапке появляется баннер «Дождь на участке М-11, км 189+445 - 194+292»;
  - на карте появляются маленькие синие точки самосвалов — они перенаправлены
    с участка №2 на ближайший сухой (№3 «км 194+292 - 199+661»);
  - в журнале решений появляются записи: «закрыто зелёное окно», 2–3
    «перенаправлен самосвал», 2 «создан наряд ТО»;
  - в карточке «Открытые наряды ТО» появляются записи на простаивающие
    машины с указанием бригады.

### Шаг 4. Реакция расчётов (30 сек)
- Возвращаемся на карточку участка №2 — таймлайн перерисовался: первые часы
  красные, «зелёного окна нет», виджет тоннажа показывает 0 т.
- Кликаем «Сброс» в правой панели — состояние возвращается к исходному.

### Шаг 5. Что считается реально (30 сек)
Подчёркиваем, что под капотом — формулы из техкарты:
- скорость укладчика `V = П_абз / (a · h · ρ · 60)` (формула 2);
- максимальный тоннаж с учётом времени доставки и остывания смеси до 80 °C
  по таблице 1 техкарты;
- «зелёное окно» с проверкой осадков (`precip_mm/h`, `precip_probability`) и
  температуры (`≥ +5 °C`, `≥ +10 °C` для тонких слоёв);
- расстояния по гаверсинусу, время доставки = max(заказ АБЗ, поездка).

## 3. Что моковое в MVP

| Подсистема | Реализация в MVP | Что появится после хакатона |
|---|---|---|
| Метео-API | OpenWeatherMap при наличии ключа, иначе mock fallback | Ансамбль OpenWeather/Tomorrow.io/Open-Meteo |
| База данных | in-memory `DemoState` на JSON-фикстурах | PostgreSQL + PostGIS + TimescaleDB по плану в `docs/data_model.md` |
| GPS самосвалов | синтетические точки на карте между АБЗ и участком | Реальный поток координат через WebSocket |
| Дождь | ручной триггер через `POST /demo/sudden-storm` | Автоматический триггер по nowcast-радарам |
| Карта | OSM-тайлы + актуальный RainViewer-радар из metadata API | Тайлы Яндекса/2GIS + локальный кэш |
| Аутентификация | отсутствует | JWT/Keycloak в проде |

## 4. Что считается реальным уже сейчас

- Все формулы из техкарты в [backend/app/services/algorithms/](backend/app/services/algorithms/).
- Таблица 1 «продолжительность уплотнения» в [compaction_table.yaml](backend/app/services/algorithms/compaction_table.yaml).
- FSM статусов самосвала в [truck_fsm.py](backend/app/services/logistics/truck_fsm.py).
- Геометрия — расстояния по большому кругу, реальные пикетажи участков
  М-11 (`184+807 - 209+311`) и приближённые координаты для карты.
- Погодный endpoint `/api/v1/sites/{id}/forecast` сначала пробует OpenWeatherMap,
  затем автоматически откатывается на mock fallback.
- 20+ backend-юнит-тестов (`backend/.venv/Scripts/python.exe -m pytest`).

## 7. Известные ограничения MVP

- Анимация перемещения самосвалов условная — точка ставится в середине отрезка
  АБЗ → участок. Этого достаточно для показа.
- Время «сейчас» — реальное системное; чтобы окно всегда выглядело красиво,
  таймлайн рассчитывается на ближайшие 24 часа.
- При нескольких подряд запусках «дождя» наряды ТО накапливаются — это
  специально, чтобы можно было показать длинный журнал.
- Кодировка кириллицы в PowerShell-консоли может отображаться некорректно,
  но HTTP-ответы клиенту приходят корректным UTF-8.

## 8. Если что-то пошло не так

| Симптом | Решение |
|---|---|
| Порт 8000 занят | `Get-NetTCPConnection -LocalPort 8000 \| ForEach { Stop-Process -Id $_.OwningProcess -Force }` |
| Карта пустая | проверьте интернет — тайлы OSM/RainViewer загружаются с CDN |
| На карте нет радара | RainViewer metadata временно недоступна; MVP скрывает слой вместо битых тайлов |
| Погода показывает `mock fallback` | проверьте `OPENWEATHER_API_KEY` в локальном `.env` или лимит OpenWeatherMap |
| `npm run dev` не стартует | удалить `frontend/node_modules` и переустановить |
| Прогноз всегда «нет окна» | у текущего часа `dt` смесь не успевает доехать; нажмите Сброс и подождите минуту |
| Кнопки не работают | проверьте, что backend на 8000 и в DevTools нет CORS-ошибок |
