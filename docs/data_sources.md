# Источники метео-данных

> **Владелец документа: E2 (инженер данных).**

## 1. Требования

| Параметр | Минимум | Желательно |
|---|---|---|
| Горизонт прогноза | 24 ч | 72 ч |
| Шаг прогноза | 1 ч | 15 мин |
| Тип осадков | мм/ч + флаг rain/snow | + интенсивность по слоям |
| Температура | T возд, 2 м | + T росы, T основания |
| Ветер | скорость + порыв | + направление |
| Радар | прямоугольные тайлы | nowcasting на 30 мин |
| Лимит free tier | ≥ 1000 запросов/день | ≥ 60 запросов/мин |

## 2. Кандидаты

### 2.1 OpenWeatherMap
- Endpoint: `api.openweathermap.org/data/3.0/onecall`
- Free: 1000 запросов/день, 60/мин.
- Возвращает hourly на 48 ч, daily на 8 дней.
- Документация: https://openweathermap.org/api/one-call-3
- **Используем как основной источник.**

### 2.2 Tomorrow.io
- Endpoint: `api.tomorrow.io/v4/weather/forecast`
- Free: 500 запросов/день, 25/час, 3/сек.
- Сильная сторона: minutely-прогноз на 1 ч (для «успеть до дождя» отлично).
- Документация: https://docs.tomorrow.io
- **Используем для краткосрочной коррекции.**

### 2.3 Gismeteo (через open-meteo как fallback)
- Платный официальный API. Альтернатива: **Open-Meteo** — бесплатно, без ключа,
  агрегирует множество моделей (ICON, GFS, ECMWF).
- Endpoint: `api.open-meteo.com/v1/forecast`
- **Используем для ансамбля без затрат.**

### 2.4 RadarMap / OSM radar tiles
- Тайлы радара осадков, например RainViewer:
  `tilecache.rainviewer.com/v2/radar/{time}/{size}/{z}/{x}/{y}/{color}/1_1.png`
- API списка тайлов: `api.rainviewer.com/public/weather-maps.json`
- Бесплатно, без ключа.
- **Только для визуализации на фронте, в расчёт не идёт.**

```json
{
  "site_id": 5,
  "lat": 56.123,

  "source": "ensemble",

  "precip_probability": 0.18,
  "precip_type": null,
```

## 5. Ансамбль

Базовая формула (взвешенное среднее):

```
value_ens = Σ wᵢ · valueᵢ / Σ wᵢ
```

- `wᵢ` — обратная MAE источника `i` за последние 14 дней по этой локации.
- Для бинарных полей (rain/snow) — мажоритарное голосование.
- Если 1 источник не отвечает > 2 минут → исключается, нормализуем веса.

(Опц.) **Калибровка**: линейная регрессия `Scikit-learn` или градиентный бустинг
`XGBoost` на исторических разницах «прогноз − факт».

## 6. Резервы и устойчивость

- **Кэш Redis** на 5 минут — снимает 80 % внешних запросов.
- **VCR-кассеты** для тестов: фикстуры в `backend/tests/cassettes/`.
- **Mock-провайдер** `services/weather/mock.py` для демо без интернета.
- **Деградация**: если все источники упали — отдаём последний удачный прогноз
  с пометкой `stale=true`.
