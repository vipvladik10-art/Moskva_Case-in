Длительность: **8 недель = 4 спринта по 2 недели**.
Каждый спринт оканчивается демо и ретроспективой.

Обозначения: **P1, P2** — программисты; **E1, E2, E3** — инженеры (см. [TEAM.md](./TEAM.md)).

## Спринт 2 — Алгоритмическое ядро (недели 3–4)

**Цель спринта**: система умеет считать «зелёное окно» и максимальный тоннаж
для каждого участка.

### Алгоритмика (E1 + P1)
- [ ] **«Зелёное окно»** (E1: спецификация → P1: код):
      агрегированный прогноз + время доставки `t_доставки = расстояние / средняя скорость`
      → интервал `[t_старт, t_конец]`, в котором:
      - вероятность осадков ≤ порога (по умолчанию 30 %),
      - T возд ≥ +5 °C (≥ +10 °C для тонких слоёв),
      - длительность ≥ минимально необходимой для захватки.
- [ ] **Калькулятор «Успеть до дождя»** (E1 → P1):
      `V_max = ПУ_укладчика · ширина · толщина · ρ · t_окна`,
      с проверкой по производительности АБЗ (`П_абз / (a·h·ρ·60)`).
- [ ] **Температурная модель остывания** (E1 → P1): интерполяция таблицы 1
      техкарты, выдаёт допустимое время уплотнения.
- [ ] Unit-тесты на справочных примерах (P1, ревью E1).

### Ансамбль прогнозов (E2)
- [ ] Алгоритм объединения: медианный/взвешенный по исторической точности.
- [ ] Поле `confidence` в ответе API.
- [ ] (Опц.) baseline-модель `scikit-learn` для коррекции прогноза.

### API (P1)
- [ ] `POST /api/v1/sites/{id}/green-window` — расчёт окна.
- [ ] `POST /api/v1/sites/{id}/max-tonnage` — расчёт тоннажа.
- [ ] `GET /api/v1/sites/{id}/forecast` — агрегированный прогноз на 24 ч.

### Фронтенд (P2)
- [ ] Страница участка: график прогноза, выделение «зелёного окна» на таймлайне.
- [ ] Виджет «Сколько тонн заказать сейчас».
- [ ] Тосты и состояния загрузки/ошибок.

### QA (E3)
- [ ] Сценарии в `pytest-bdd` или просто `pytest`: «дождь через 2 часа, что считает система».
- [ ] Mock-провайдер погоды для воспроизводимых тестов.

**Демо S2**: для участка №3 на М-11 система говорит: «зелёное окно
14:20–17:40, можно заказать до 84 т смеси».

## Спринт 3 — Логистика, ресурсы, ТО (недели 5–6)

**Цель спринта**: оперативное управление в реальном времени.

### Логистика (P1 + E1)
- [ ] Модель `Truck` со статусом FSM: `loading → en_route → waiting → unloading → idle`.
- [ ] WebSocket `/ws/trucks` для онлайн-позиций (для демо — симулятор).
- [ ] **Динамическое перенаправление**: при поступлении радар-сигнала «дождь на участке A»
      сервис находит ближайший сухой участок B и подменяет точку назначения
      загруженных самосвалов. Запись истории решений в `truck_dispatch_log`.

### Планировщик ТО (P1 + E1)
- [ ] Справочник машин (укладчики, катки) и регламентов ТО (по моточасам/дням).
- [ ] Триггер: при `precipitation > 0` и `idle_machines > 0` создаётся `MaintenanceTask`.
- [ ] Эндпоинт `GET /api/v1/maintenance/tasks?status=open`.

### Подвоз тех-смесей (P1 + E1)
- [ ] Правила: T возд < 0 → антифриз; начало смены — антиадгезионка (керосин).
- [ ] Сервис генерирует заявки `SupplyOrder` и привязывает к АБЗ/участку.

### Фронтенд (P2)
- [ ] Слой карты «Самосвалы» с цветовой индикацией статуса.
- [ ] Панель «Перенаправления» — список последних решений системы.
- [ ] Экран «Наряды механикам» (mobile-friendly).
- [ ] Push-уведомления о закрытии «зелёного окна» (Web Push, демо-режим).

### Данные (E2)
- [ ] Симулятор движения самосвалов (для демо без реальных GPS).
- [ ] Симулятор изменения погоды (можно «двигать дождь» по карте).

### QA / DevOps (E3)
- [ ] Интеграционный сценарий «полный цикл»: заказ → дождь → перенаправление → ТО.
- [ ] Метрики: время отклика API, throughput WebSocket.

**Демо S3**: запускаем симулятор, на 7-й минуте «начинается дождь» на участке №2 —
система перенаправляет 3 самосвала на участок №5 и выписывает 2 наряда на ТО.

## Риски и митигация

| Риск | Вероятность | Митигация | Owner |
|---|---|---|---|
| Лимиты бесплатных метео-API | Высокая | Кэш + 3 источника + mock-режим | E2 |
| Сложность PostGIS-запросов | Средняя | Готовые рецепты, индексы GIST | P1, E3 |
| Интеграция фронт↔бэк зависит от готовности API | Средняя | Контракт OpenAPI заранее (Спринт 1) | P1, P2 |
| Несоответствие алгоритмов ГОСТ | Высокая критичность | E1 ревьюит ВСЕ PR с формулами | E1 |
| Не хватит времени на ML | Средняя | ML помечен как «опционально» | E2 |
| Демо упадёт онлайн | Низкая | Локальный запуск + видеозапись | E3 |
